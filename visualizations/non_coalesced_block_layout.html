⚠️ The Critical Issue
Each warp's 32 threads span 2 rows!
Threads shown as (threadIdx.x, threadIdx.y)
Since blockDim.x = 16, threads (0-15, 0) are in row 0, and threads (0-15, 1) are in row 1.
This means they access memory locations 1024 elements apart!

❌ Non-Coalesced Access
The GPU cannot combine these into a single memory transaction because threads access locations 1024 elements apart. Result: 2 separate transactions per warp, 50% bandwidth waste.
